Justificación de Recursos — Proyecto La Casa del Pastel

Este documento adapta la planificación de recursos para el proyecto La Casa del Pastel, considerando que la primera fase de despliegue se realizará en InfinityFree (hosting gratuito). La idea es lanzar un prototipo público con catálogo, gestión de pedidos y panel de usuario, y validar UX/negocio antes de migrar a un servicio de pago en caso de crecimiento.

Resumen técnico y objetivos

Objetivo: disponer de un sitio público con catálogo de productos, carrito y panel de usuario para clientes (funcionalidad CRUD básica en la base de datos). Alcance inicial aproximado: catálogo de ~50 productos, páginas estáticas y dinámicas (login, panel, carrito), y un volumen moderado de tráfico de pruebas y clientes reales.

Por qué InfinityFree encaja con La Casa del Pastel (justificación)

  • Bajo coste / rápido despliegue: al ser gratuito permite publicar el prototipo sin inversión inicial y validar la experiencia de compra (catálogo, carrito, login) con clientes reales.
  • Recursos suficientes para la fase 1: la combinación de 5 GB de disco y ancho de banda ilimitado cubre los activos del sitio (imágenes optimizadas, CSS/JS y páginas) y visitas moderadas mientras la marca gana tracción.
  • Entorno técnico moderno: soporte para PHP 8.x y MySQL/MariaDB actuales facilita portar el desarrollo local (XAMPP) sin reescrituras.
  • Escalado gradual y control de riesgos: si el sitio crece en tráfico o complejidad (pagos, integraciones, muchos productos), la arquitectura se puede migrar a VPS o cloud con balanceo y CDN. Mientras tanto, InfinityFree sirve como entorno de validación y pruebas A/B.

Requisitos técnicos adaptados al proyecto

  1. Hosting web (InfinityFree): alojar todos los archivos públicos HTML/CSS/JS/PHP. Usar FTP para subir archivos grandes (>10MB) y mantener la raíz en /htdocs.
  2. Base de datos MySQL: una base de datos para la aplicación (usuarios, productos, pedidos). Mantener índices y consultas optimizadas para evitar sobrecargar el servicio (fair-use).
  3. Optimización de activos: imágenes comprimidas (WebP/avif opcional), sprites o lazy-loading para reducir I/O y mejorar tiempos de carga.
  4. Importación de BD: si el volcado (.sql) excede el límite de importación web, dividir el archivo o importar por partes desde phpMyAdmin, o bien crear la BD vacía y subir datos por scripts/TUI por FTP.
  5. Seguridad básica: SSL (InfinityFree genera certificados gratuitos), validación de inputs en servidor y uso de prepared statements para prevenir inyecciones SQL.
  6. Plan de respaldo y migración: mantener copia local del código y BD; si la tienda crece, planificar migración a VPS/Cloud con CDN y backups programados.

Riesgos y limitaciones (y mitigación)

  • Límite de tamaño de archivos: la carga directa tiene límites (~10 MB). Mitigación: subir por FTP, optimizar imágenes y evitar backups pesados vía php. Esto es relevante para plantillas grandes o backups.
  • Fair-use en MySQL: no hay límite duro de tamaño, pero sí límites por uso intensivo de CPU/consultas. Mitigación: queries eficientes, paginación, caché en cliente y limitar operaciones costosas.
  • Sin SLA empresarial: hosting gratuito no ofrece garantía de disponibilidad ni soporte prioritario. Mitigación: planificar migración a hosting pago si se llega a tráfico o facturación significativa.
