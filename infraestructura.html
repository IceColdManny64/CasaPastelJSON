Justificación de Recursos del Proyecto

Para el proyecto EcoTienda MX, se definió una infraestructura hipotética que permite iniciar operaciones con un catálogo de tamaño mediano y tráfico moderado. Cada recurso fue seleccionado considerando un balance entre rendimiento, seguridad y costos.

Componentes empleados

  • Cliente/Usuario (Navegador): Permite acceso universal desde PC o móvil mediante Chrome, Firefox u otros navegadores.
  • Internet / ISP: Canal público de comunicación que conecta al cliente con el servidor. Puede ser red fija o móvil (4G/5G).
  • CDN (opcional): Mejora la velocidad de carga y reduce la carga del servidor principal en campañas con alto tráfico.
  • Firewall / WAF básico: Protege contra accesos maliciosos y ataques comunes como inyecciones o bots automatizados.
  • Servidor Web / Hosting: Aloja los archivos HTML, CSS y JS del sitio. En este escenario solo se considera contenido estático.
  • Sistema Operativo y Servidor: Linux Ubuntu junto con Nginx o Apache, tecnologías probadas, estables y de bajo costo.
  • Recursos de hardware: 2 vCPU, 4 GB RAM, 50 GB SSD y 1 TB de ancho de banda. Estos parámetros son suficientes para un inicio estable de una PyME.

Conclusión

La elección de estos recursos responde a la necesidad de ofrecer un sistema accesible, seguro y con capacidad suficiente para un negocio en etapa inicial. Además, la arquitectura es flexible: puede crecer a un VPS más robusto o a la nube con balanceo y CDN a medida que la demanda aumente.
